### Phase 2 — /team-Kurz-URL via next.config.ts

Der lange Pfad `/konzept/leaderinnen-vorschau.html` wäre für WhatsApp-
Verbreitung umständlich. Erster Versuch als Array-Rewrite (afterFiles-
Semantik) lieferte falschen Content — die dynamische `[partner]`-Route
matched „team" als Beraterin-Slug.

Fix mit `beforeFiles`-Object-Form:
```typescript
async rewrites() {
  return {
    beforeFiles: [
      { source: "/team", destination: "/konzept/leaderinnen-vorschau.html" },
    ],
    afterFiles: [], fallback: [],
  };
}
```

Damit läuft der Rewrite VOR der Dynamic-Route-Auflösung. Live-Test:
`myglowmatch.de/team` liefert HTTP 200 mit dem Vorschau-Landing-Content.

## Konventionen hinzugekommen (für spätere Sessions)

- **n8n-Änderungen per Python-Skript, nicht per Anleitung.** Thomas'
  klare Ansage: „du hast doch bisher in n8n auch alles selbst gemacht".
  Ergo: bei Bau-Aufgaben in n8n immer Skript im Muster von
  `sync_rules_to_workflow.py` / `patch_webhook_respond.py`. Muster:
  api_call-Helper, load_env, strip_readonly, Backup vor Deploy,
  --dry-run als Option, GET-Verifikation nach PUT. Idempotenz per
  „Node bereits vorhanden?"-Check, damit Skript beliebig oft laufen
  kann.

- **Next.js-Rewrites vor Dynamic Routes: `beforeFiles` benutzen.** Ein
  Array-Return von `rewrites()` wird als `afterFiles` interpretiert und
  läuft NACH Dynamic-Route-Matching. Wenn `/team` von einer `[partner]`-
  Route abgefangen würde, muss der Rewrite explizit als
  Object-Return mit `beforeFiles: [...]` deklariert werden. Merken:
  `beforeFiles` für Rewrites gegen Dynamic Routes, `afterFiles` für
  Rewrites gegen statische Files.

- **Wenn ein Element beim ersten Bau invisible ist, ist es meistens ein
  CSS-Konflikt zwischen Tailwind-Klassen und Inline-Styles.** Die
  Nächste-Schritte-Nummerierung verschwand, weil Tailwind `list-decimal`
  von einem Inline-Style-Reset überschrieben wurde. Fix: explizit
  `listStyleType: "decimal"` + `listStylePosition: "outside"` +
  `paddingLeft` in den Inline-Styles setzen — dann greift es zuverlässig.
  Muster für alle inline-gestylten Listen.

- **Chrome-Headless-Screenshot für OG-Images.** Statt Bildbearbeitung:
  HTML-Vorlage in 1200×630 bauen, dann Chrome mit
  `--headless=new --window-size=1200,630 --screenshot=<pfad>` rendern
  lassen. Ergebnis ist JPEG unabhängig vom Dateinamen (Content-Type
  passt bei .jpg-Endung sofort). Reproduzierbar, unter Versionskontrolle.

- **WhatsApp-Preview-Cache ist aggressiv.** Nach jedem OG-Image-Update
  kann es 5–15 Minuten dauern bis WhatsApp die neue Kachel zieht.
  Facebook Sharing Debugger oder Twitter Card Validator triggern den
  Refresh sofort. Bei Testerinnen-Tests: erst warten oder erzwingen,
  dann versenden.
