### Skript-Design und -Konvention

`sync_rules_to_workflow.py` (~290 LOC final):
- **Sheet-Read**: gspread über `sheets_writer.open_sheet()` + `get_all_values()`, alles roh als String. Leere Zeilen werden gefiltert (keine gspread-Trailer-Rows in embedded JSON).
- **Node-Patch**: SYNC-Region zwischen `// >>> SYNC:{tab} — synced {ts} — {N} rows —` und `// <<< SYNC:{tab}`. Erster Sync ersetzt `init_pattern` (die aktuelle `$items(...)`-Zeile), Folgeaufrufe finden Marker-Region und ersetzen sie idempotent.
- **Emit-Stile**: `"stmt"` (ersetzt komplette `const X = $items(...).map(item => item.json);` — Node 14) und `"expr"` (ersetzt nur `$items(...).map(...)`-Sub-Ausdruck, umgebende filter-Chain bleibt intakt — Node 04, Node 11).
- **`post_init_replace`**: optionale Liste von `(regex, replacement)`-Paaren, die einmalig beim init-Mode zusätzlich zur Sync-Region angewandt werden. Für Node 11 nötig, weil `.map(it => it.json)` dort in getrennter `const rules = ruleItems.map(...).filter(...)`-Deklaration nach `ruleItems` steht — muss beim init weg, sonst würden Plain-Dicts auf `.json` mappen und leere Regeln erzeugen.
- **API-Handling**: analog `patch_retry_sheets_loaders.py` (Migration #25) — Top-Level- und Settings-Cleanup vor PUT. Backup-Konvention: `backups/workflow_sync/workflow_backup_YYYYMMDD_HHMMSS_pre_sync.json`.
- **GET-Verifikation**: prüft dass Marker-Header im Live-Workflow steht und row-count übereinstimmt (aus Marker-Header-Kommentar `— N rows —`, robust gegen emit_style-Varianten).
- **Dry-Run-Modus**: `--dry-run` schreibt Preview in `backups/workflow_sync/workflow_dryrun_YYYYMMDD_HHMMSS.json`, kein PUT.

## Konventionen — neu

- **K-15 (implizit, für nächste Session zu memorieren)**: Sync-Nodes müssen nach expr-Sync auf `.map(it => it.json)`-Reste in ANDEREN Statements geprüft werden. Wenn Loader-Zuweisung an Zwischenvariable + separate filter-Deklaration mit `.map()`-Kette → `post_init_replace` nötig, sonst semantisch tot. Pre-Sync-Kontroll-Check: `grep '.map(it => it.json)'` im Node-jsCode nach Init-Pattern-Ersetzung. Ich habe das beim Node 11 im Dry-Run sichtbar gemacht bevor PUT — Prozess-Konvention.
- **Verwaiste Sheets-Loader-Nodes** verbrauchen weiter Google-Quota und crashen weiter, obwohl ihr Output nicht mehr konsumiert wird. Nach dem letzten Sync eines Sheets muss der Loader-Node aus dem Workflow entfernt werden. Sonst rechnet der Bulk-Test noch mit dem alten Quota-Fußabdruck.
